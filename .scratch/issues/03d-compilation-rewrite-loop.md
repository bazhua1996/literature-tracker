# 03d — 编译质量反馈与重试

**What to build:** 审校不满意时在追踪器内触发返工，选择返工模式（全量重译 / 指定章节重译 / 换提示词风格），追踪器自动更新对应 `.md` 文件。

**Blocked by:** 03c — 需要一键 AI 编译能力作为重译的底层引擎

**Status:** ready-for-agent

- [ ] 追踪器对"编译中"状态的论文显示"🔄 返工"下拉菜单
- [ ] 返工模式 1：全量重译 — 换提示词风格（政策型/学术型/简报型），重新调用 AI 生成全部章节
- [ ] 返工模式 2：章节重译 — 选择指定章节（如一、四、八），仅重写选中章节，保持其他章节不变
- [ ] 返工模式 3：智能修订 — 用户输入修改意见（如"数据部分太干，增加解释"），AI 按意见调整全文
- [ ] 返工操作记录写入 YAML frontmatter `revision_history` 字段（时间戳 + 操作类型）
- [ ] 状态管理：已定稿 → 返工后自动回退到"编译中"
- [ ] 验证：对已定稿编译稿执行全量重译 → 确认全部章节被替换、revision_history 追加记录、状态变为编译中
