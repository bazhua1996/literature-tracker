```markdown
---
status: 已发现

  title: "原文标题"
  date: "YYYY-MM-DD"
  source: ECB
  authors: "作者"
  paper_type: Report
  detail_url: "https://..."
  pdf_path: "Attachments/PDFs/xxx.pdf"
  report_number: "EBA/REP/2025/40"
tags: [ECB, 支付欺诈, ...]
created: YYYY-MM-DD
---

> [原文标题] | [机构] | [发布日期]

**摘要：** [150-200字概述：背景+核心数据+关键发现+政策启示]

**关键词：** [3-5个中文关键词]

## 一、引言与背景
[报告的发布背景、数据来源、方法论、覆盖范围、局限性]

...

## 八、结论与政策启示
[综合结论 + 结合中国国情的政策建议（3-5条）]
```

## 二、格式规范

| 维度 | 规范 |
|------|------|
| 标题 | `# 关于[来源]《[报告名]》的分析` |
| 引用行 | `> 原文标题 \| 机构 \| 日期` |
| 章节编号 | 中文数字：一、二、三…八 |
| 子节编号 | 括号中文数字：（一）（二）（三） |
| 摘要 | 150-200字，**不编号**，置于关键词之前 |
| 关键词 | 3-5个，分号分隔 |
| 正文段落 | 首行不缩进（Markdown），段落间空行 |
| 数据呈现 | 融入正文叙事，不单独列表；关键数字用**粗体**强调 |
| 英文术语 | 首次出现标注全称+缩写，后续用缩写 |
| 长度 | 3500-5000 中文字符 |

## 三、内容规范

### 3.1 写作风格
- **学术分析体**：不是翻译，是深度分析报告
- **数据叙事**：数字不在摘要中堆砌，而是在分析段落中自然引用并解读其含义
- **视角定位**：始终站在中国政策制定者和行业研究者的视角
- **术语保持**：关键英文术语首次出现时标注原文

### 3.2 结构逻辑
```
摘要（全景鸟瞰）
  ↓
引言（背景+数据来源+局限）
  ↓
总体规模（金额+笔数+结构+趋势）
  ↓
手段/渠道演变（远程vs非远程+欺诈类型）
  ↓
制度分析（SCA有效性+豁免风险）
  ↓
法律经济分析（损失分担+国别差异）
  ↓
跨境风险（地缘敞口）
  ↓
国别对比（关键国家数据）
  ↓
结论+政策启示（对中国建议）
```

### 3.3 数据引用原则
- 绝对数值 + 同比变化 + 相对比率三位一体
- 横向对比（工具间、国家间、远程vs非远程）
- 时间趋势（H1 2022 → H2 2024）
- 结构性解读（不仅列数字，解释数字背后的驱动因素）

### 3.4 与中国关联
- 每份编译报告都应有"政策启示"章节
- 结合中国实际：监管框架、市场特征、发展阶段
- 不是简单照搬，而是分析"可借鉴性"

## 四、与旧模板的关键差异

| 维度 | 旧模板 | 新模板 |
|------|--------|--------|
| 标题 | `# [原文标题中文译名]` | `# 关于[来源]《[报告名]》的分析` |
| 前端 | 双语关键引述 | **移除**，数据融入分析叙事 |
| 核心内容 | 五段式（背景/观点/数据/信号/展望） | **灵活章节**（按报告逻辑 6-8 章） |
| 末尾 | 标签 | 关键词 + 标签 |
| 定位 | 翻译+摘要 | **深度分析报告** |
| 产出 | 1×md + 2×docx | 1×md（分析报告） + 1×docx（排版输出） |
